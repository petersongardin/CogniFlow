# 23 — App próprio, alertas, recompensas e tela de capa

## 1. Princípio

O CogniFlow pode começar por WhatsApp ou interface simples, mas a experiência final ideal deve ser um app próprio.

O motivo é funcional: pessoas com TDAH respondem melhor a estímulos pequenos, visuais, sonoros, recorrentes e contextualizados. O WhatsApp é útil para MVP, mas não oferece controle suficiente sobre experiência, notificações, tela inicial, reforço visual, sons, recompensas e ritual de execução.

## 2. Inspiração de produto

O Duolingo é uma referência importante por usar:

- alertas visuais;
- alertas sonoros;
- sequência diária;
- recompensas pequenas;
- sensação de progresso;
- retorno frequente;
- linguagem simples;
- reforço imediato;
- fricção baixa para retomar.

O CogniFlow deve aprender com esse tipo de mecânica, mas sem copiar cegamente e sem transformar execução vital em jogo vazio.

## 3. Diferença essencial em relação ao Duolingo

Duolingo recompensa prática voluntária de aprendizado.

CogniFlow precisa apoiar tarefas vitais que muitas vezes são aversivas, obrigatórias, atrasadas ou emocionalmente carregadas.

Portanto, o reforço do CogniFlow deve ser:

```text
recompensa saudável por avanço real
não dopamina barata que substitui execução
```

## 4. App como capa do celular

O CogniFlow pode evoluir para uma experiência parecida com uma camada de capa do celular.

Objetivo:

- mostrar a Minitaf vital atual;
- lembrar a consequência da não execução;
- mostrar a recompensa boa da execução;
- impedir que o usuário esqueça o que é prioritário;
- reduzir fuga para apps de dopamina barata;
- funcionar como tela de retorno ao foco.

Conceito:

```text
Ao desbloquear o celular, o usuário deve reencontrar a tarefa que protege sua vida real.
```

## 5. Notificações sonoras e visuais

O app deve suportar:

- som curto de início de Minitaf;
- som de follow-up;
- vibração;
- notificação visual com a Minitaf atual;
- badge de tarefa vital pendente;
- alerta de prazo;
- alerta de sono;
- alerta financeiro;
- lembrete de pausa;
- reforço positivo após conclusão.

## 6. Tipos de alerta

### 6.1 Alerta de início

```text
Sua Minitaf vital está pronta: abrir o relatório e escrever os títulos principais.
```

### 6.2 Alerta de prazo

```text
Essa entrega vence em breve. Uma Minitaf agora reduz o atraso.
```

### 6.3 Alerta antidopamina

```text
Você está fora da tarefa vital. Volte para a Minitaf atual.
```

### 6.4 Alerta de sono

```text
Agora a tarefa vital é proteger o sono. Amanhã sua execução depende disso.
```

### 6.5 Alerta financeiro

```text
Seu mês está projetado para fechar negativo. Evite novo gasto não essencial.
```

## 7. Recompensas saudáveis

O sistema deve reforçar progresso real.

Exemplos:

- sequência de dias com Minitaf vital concluída;
- número de Minitafs concluídas na semana;
- redução de atraso;
- tarefa profissional protegida;
- sono respeitado;
- gasto evitado;
- dívida organizada;
- projeto retomado.

## 8. Recompensa boa versus dopamina barata

O app deve reforçar ações que protegem a vida real.

Não deve recompensar apenas uso do app, conversa longa ou planejamento infinito.

Regra:

```text
A recompensa deve vir por execução concluída, não por ficar conversando com o sistema.
```

## 9. Sistema de sequência diária

O app pode ter sequência diária, mas deve evitar punição emocional.

Sequência sugerida:

```text
Dias com pelo menos 1 Minitaf vital concluída.
```

Se o usuário falhar, o sistema deve oferecer reentrada sem culpa:

```text
A sequência caiu, mas a retomada começa com uma Minitaf pequena agora.
```

## 10. Tela principal do app

A tela principal deve mostrar apenas o essencial:

```text
Minitaf vital atual
Consequência se não executar
Recompensa se executar
Botões: iniciar / acabei / travei / adiar
```

Evitar:

- dashboard complexo no início;
- listas enormes;
- múltiplas prioridades simultâneas;
- excesso de métricas;
- recompensas visuais que virem distração.

## 11. Detecção do comportamento descrito pelo usuário

O sistema deve reconhecer quando o usuário está fazendo exatamente o que quer evitar.

Exemplo real:

```text
O usuário está gostando de conversar sobre o app, porque isso gera dopamina, enquanto existe uma tarefa vital atrasada que precisa ser executada.
```

O comportamento esperado do CogniFlow é interromper gentilmente a conversa e redirecionar:

```text
Você está recebendo dopamina planejando o sistema. A tarefa vital agora é o relatório. Execute a próxima Minitaf.
```

## 12. Sintomas corporais de resistência

O app deve registrar que, ao iniciar tarefa aversiva, o usuário pode sentir:

- coceira;
- dor de cabeça;
- ardência nos olhos;
- inquietação;
- vontade de fugir;
- irritação;
- cansaço repentino.

O sistema deve responder assim:

```text
Isso pode ser resistência de iniciação. Não trate como ordem para parar. Reduza para a menor Minitaf possível.
```

## 13. WhatsApp versus app próprio

### WhatsApp

Bom para:

- MVP rápido;
- teste individual;
- áudio simples;
- baixa barreira inicial.

Limitações:

- notificações pouco controláveis;
- risco de mistura com distrações;
- pouca experiência visual;
- sem controle profundo de tela inicial;
- menos adequado para recompensa e sequência.

### App próprio

Bom para:

- experiência sensorial;
- alertas controlados;
- tela de foco;
- recompensas visuais;
- controle de fluxo;
- dashboard futuro;
- créditos de IA;
- privacidade e consentimento;
- onboarding multiusuário.

## 14. Fases de implementação

### Fase 1 — MVP conversacional

- WhatsApp ou web chat;
- Minitafs;
- follow-up;
- banco;
- créditos;
- tarefas vitais.

### Fase 2 — Web app/PWA

- tela principal de Minitaf;
- notificações web;
- login;
- histórico;
- preferências;
- créditos.

### Fase 3 — App mobile

- notificações nativas;
- sons;
- vibração;
- sequência diária;
- tela de foco;
- widgets;
- modo capa/atalho de celular.

## 15. Critério de sucesso

A funcionalidade é bem-sucedida quando o app consegue:

1. puxar o usuário de volta da dopamina barata;
2. lembrar a Minitaf vital atual;
3. usar som, visual e recompensa para sustentar execução;
4. reforçar avanço real sem virar distração;
5. proteger sono, trabalho, saúde e finanças;
6. reduzir a distância entre saber o que fazer e iniciar.
